# 과제 안내

## 배경

최근 팀원 퇴사로 인수인계가 불완전하게 됐습니다. 코드는 돌아가는 것처럼 보이지만, 이 파이프라인의 결과를 신뢰하고 실제 투자 의사결정에 쓸 수 있는지 확신하기 어렵습니다.

이 시스템은 FastAPI 서버, 백테스트 워커, Redis로 구성된 분산 파이프라인입니다. 코드를 넘겨받으셨습니다. **이 파이프라인을 프로덕션에서 신뢰할 수 있는 상태로 만들어 주세요.**

---

## 제출

**필수**
1. Git repo (private) — 커밋 히스토리 포함. collaborator 초대 이메일을 안내해드립니다.
2. `REPORT.md` — 아래 항목을 포함해주세요:
   - 발견한 문제 목록과 각각의 심각도 판단
   - **고친 것과 고치지 않은 것, 그 우선순위 판단 이유**
   - 이 파이프라인의 결과를 지금 신뢰할 수 있는지, 없다면 무엇이 더 필요한지
3. `DECISION_LOG.md` — 아래 항목을 포함해주세요:
   - 사용한 AI 도구 목록 (Claude Code, ChatGPT, Copilot 등)
   - AI가 "괜찮다"고 했지만 본인이 의심하거나 직접 확인한 사례
   - AI 제안을 거부하거나 수정한 것과 그 이유
   - AI가 놓쳤는데 본인이 직접 발견한 것

**선택 (있으면 함께 제출)**
- `CLAUDE.md`, `.cursorrules` 등 AI 도구 설정 파일 — 작업 환경을 어떻게 셋업했는지 볼 수 있습니다.
